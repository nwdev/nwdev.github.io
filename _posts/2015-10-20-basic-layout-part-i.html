1. Adding Bootstrap to Jekyll

So my situation is the following: So far I did not really care about responsive frameworks. I know that the 2 most popular ones are ZURB Foundation and Bootstrap. That's it. So how I am supposed to choose between the two? Here are some bullet points that helped me choose:

  • My front-end skill is non-existent, so I need the framework to be relatively easy to work with. I think both of these framework are easy to use.
  • Still, I am in need of tutorials, forums. From that point of view, Bootstrap might have an edge.

But then again, I am not an expert of the topic, so I am just going for Bootstrap, because It seems the safer choice for now.

Following the instructions here, I added these lines to head.html (./_includes/head.html):


    <!-- Latest compiled and minified CSS -->
    <link rel="stylesheet" href="https://maxcdn.bootstrapcdn.com/bootstrap/3.3.5/css/bootstrap.min.css">
    <!-- Optional theme -->
    <link rel="stylesheet" href="https://maxcdn.bootstrapcdn.com/bootstrap/3.3.5/css/bootstrap-theme.min.css">
    <!-- bootstrap.js needs jQuery -->
    <script src="https://ajax.googleapis.com/ajax/libs/jquery/1.11.3/jquery.min.js"></script>
    <!-- Latest compiled and minified JavaScript -->
    <script src="https://maxcdn.bootstrapcdn.com/bootstrap/3.3.5/js/bootstrap.min.js"></script>

2. favicon

Okay, so I thought adding a favicon to the site would take about a minute.
Well I was wrong, it took 10 minutes.(wut?)
Okay let me explain.
I thought adding a favicon to the site would mean getting an image, converting it to a proper size(using a tool like: this), putting it in the root folder named favicon.ico, and then adding these two lines to head.html:


    <link rel="shortcut icon" href="/favicon.ico" type="image/x-icon">
    <link rel="icon" href="/favicon.ico" type="image/x-icon">

Well nope. At least not on Firefox. I tried it on Chrome, it worked just fine. I don't know the exact reasons, but these 2 stackoverflow questions suggest few solutions:

  • First one
  • Second one

I went with the "trailing '?'" solution. And that's that, favicon works as expected.

3. highlight.js

Okay in my opinion, this is the coolest one.
You see, the previous code highlights were possible beacuse of highlight.js.
It supports 137 languages and 65 styles, which is more than enough for me. To set up highlight.js, I only needed to add these 3 lines to head.html:


    <link rel="stylesheet" href="//cdnjs.cloudflare.com/ajax/libs/highlight.js/8.9.1/styles/default.min.css">
    <script src="//cdnjs.cloudflare.com/ajax/libs/highlight.js/8.9.1/highlight.min.js"></script>
    <script>hljs.initHighlightingOnLoad();</script>

Adding code snippets to html is easy enough:


    <pre><code class="ruby">
      def hello_highlightJS
        puts "Welcome to Missing Semicolon!"
      end
    </code></pre>

The result is:


    def hello_highlightJS
      puts "Welcome to Missing Semicolon!"
    end

Pretty awesome isn't it?
